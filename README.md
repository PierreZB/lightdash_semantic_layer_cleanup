# dbt Semantic Layer Cleanup Tool for Lightdash

This tool is a Python-based pipeline designed to help you clean up your dbt semantic layer by identifying and commenting out unused semantic objects (dimensions, metrics, additional_dimensions) that are no longer referenced in any of your Lightdash charts.

It's a common problem for dbt projects to accumulate a large number of semantic objects over time. This can make the project difficult to maintain and understand both for developers and business users. This tool helps you to safely identify and remove unused objects, making your project cleaner, faster, and easier to work with.

## How it Works

The tool works in four main steps, orchestrated by the main `0_run_cleanup_pipeline.py` script:

1.  **Analyze Semantic Layer Objects:** The first script (`1_semantic_layer_objects.py`) parses all of your local dbt YAML files to extract a complete list of all the dimensions, metrics, and their dependencies.
2.  **Analyze Chart Usage:** The second script (`2_semantic_objects_used_in_charts.py`) first asks if you want to download the latest charts from your Lightdash instance. It then parses all of your Lightdash chart files to identify which semantic objects are currently being used.
3.  **Consolidate Object Usage:** The third script (`3_semantic_layer_objects_usage.py`) consolidates the information from the first two steps to determine which fields are used directly, indirectly (as a dependency), or not at all.
4.  **Perform Semantic Layer Cleanup:** The final script (`4_semantic_layer_cleanup.py`) reads the usage analysis and begins commenting out unused fields from your dbt YAML files. It validates with `lightdash validate` each change to ensure that it doesn't break anything. If a change broke your project, it gets reverted, and tagged with a new `# BREAKING CHANGE` comment.


## Prerequisites

Before you can use this tool, you need to have the following installed:

*   Python 3.6+
*   The `dbt`, `pyyaml`, `pandas`, and `python-dotenv` Python libraries. You can install them with pip:
    ```bash
    pip install dbt pyyaml pandas python-dotenv
    ```
*   The Lightdash CLI (https://docs.lightdash.com/guides/cli/how-to-install-the-lightdash-cli)

## Setup

1.  **Place the Tool in Your dbt Project:** This tool is designed to be run from within your dbt project. Place the `semantic_layer_cleanup` folder at the root of your dbt project. Your directory structure should look something like this:
    ```
    your_dbt_project/
    ├── models/
    ├── lightdash/
    ├── dbt_project.yml
    └── semantic_layer_cleanup/
        ├── 0_run_cleanup_pipeline.py
        ├── 1_semantic_layer_objects.py
        ├── ...
    ```

2.  **Configure the Environment:** The tool uses a `.env` file to store configuration variables. Rename the `.env.example` file to `.env` and configure the following variables:

    *   `INITIAL_VALIDATION_COMMAND`: The command to run for the initial validation of the project. This should be a thorough validation and should not skip dbt compilation.
    *   `QUICK_VALIDATION_COMMAND`: The command to run for the iterative validation in Step 4. This should be a faster validation and should skip dbt compilation.
    *   `DOWNLOAD_COMMAND`: The command to run to download your Lightdash project.

    Here is an example `.env` file:
    ```
    INITIAL_VALIDATION_COMMAND=lightdash validate
    QUICK_VALIDATION_COMMAND=lightdash validate --skip-dbt-compile
    DOWNLOAD_COMMAND=lightdash download --all
    ```

## Usage

Before running the pipeline, ensure you have a backup and/or Git setup for your dbt project files.

To run the cleanup pipeline, simply navigate to the `semantic_layer_cleanup` directory in your terminal and run the main orchestrator script:

```bash
cd /path/to/your_dbt_project/semantic_layer_cleanup
python 0_run_cleanup_pipeline.py
```

The script will guide you through each step of the process, asking for your confirmation before proceeding.

### Interpreting the Output

The `outputs` directory contains the CSV files generated by each step of the pipeline. The most important file is `3_semantic_layer_objects_usage.csv`, which contains the final usage analysis. You can use this file to understand how each of your semantic objects is being used and to identify which objects are safe to remove.

### Handling Breaking Changes

If the cleanup script finds a "breaking change", it means that a field that appears to be unused is actually a dependency for another field that *is* being used. In this case, the script will not remove the field. Instead, it will add a `# BREAKING CHANGE` comment to the field in the YAML file.

You will need to manually review these fields to determine why they are causing a breaking change. It may be that they are used in a complex way that the tool is not able to detect, or it may be that they are a dependency for a field that is used in a chart that the tool was not able to parse correctly.

## Notes

This tool is designed to be as safe as possible, but it is not a substitute for careful manual review. Always be sure to review the changes that the tool makes before committing them to your project. It is also a good idea to run your dbt and Lightdash projects' test suites after running the cleanup tool to ensure that everything is still working as expected.

This tool was fully coded by Gemini CLI 2.5 Flash & 2.5 Pro.

### Known Issues
- False positives:
In my tests, the script flagged about 5% of the objects as "Not used" in the charts, while they were in use (althgouh most of the time, they were indirectly in use). So it is imperfect in its current version and could use some additional work on dependencies identification.

- False negatives:
There may be objects in the semantic layer that are not being used in charts, but were marked as "Used"; this is an unknown as I have not checked for such cases yet.

- Time to process
The script is by nature slow as it runs `lightdash validate` after each change in the semantic layer.
